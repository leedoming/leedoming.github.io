IT 현직자 PM이 영양사 팀과 요구사항을 조율하는 구조에서, 저는 데이터 수집·정제와 생성 로직 설계를 맡았습니다. 트렌드 식단과 기존 레시피 자산을 모아 검색 가능한 형태로 정리하고, 이를 바탕으로 생성형 AI 기반 식단 생성 흐름을 설계했습니다. 전달받은 요구사항을 단순 구현이 아니라 데이터 구조와 생성 로직 관점에서 다시 해석해 구체화했습니다.

결과적으로 B2C 추천 아이디어를 현업 영양사 업무 효율 개선용 식단 생성 시스템으로 전환했고, 반복적인 레시피 탐색과 조합 과정을 줄일 수 있는 구조를 만들었습니다. 이후 이 경험은 YORILAB에서 트렌드 분석과 가상 고객 반응 예측 구조로 확장되었습니다.

대학 학사행정 챗봇 구축

대표전화 응대 경험을 바탕으로, 정확한 답변과 담당 부서 연결을 함께 설계한 상담형 RAG 프로젝트

직무 연결 포인트

실제 대표전화 응대 프로세스를 AI 상담 구조에 옮겨, 반복 문의는 자동화하고 불확실한 질문은 담당 부서로 정확히 연결하도록 설계한 사례입니다. “모르면 답하지 않는다”는 원칙으로 서비스 신뢰도를 높였다는 점이 핵심입니다.

CAR 요약
C
교내 디지털인프라팀 장학생으로 2년간 대표전화 응대를 맡으며, 같은 질문이 반복되고 결국 가장 정확한 답은 담당 책임자에게서 나온다는 점을 직접 경험했습니다. 이 경험을 바탕으로 학사행정 문의를 줄이고, 잘못된 안내를 최소화하는 Slack 기반 상담형 챗봇이 필요했습니다.
A
대표전화 응대 경험을 바탕으로, AI가 모든 질문에 답하기보다 정확한 정보 제공 또는 담당 부서 연결이 이루어지도록 설계했습니다. Lambda와 Selenium으로 데이터를 수집하고 RAG 구조를 구성했으며, 실제 대표전화 프로세스처럼 문의 성격에 따라 담당 부서와 매칭되도록 설계해 할루시네이션 위험을 줄였습니다.
R
실제 대학 Slack 워크스페이스에 배포되었고, 인턴 14일 차에 메인 챗봇 프로젝트 PM으로 지정되었습니다. 반복 문의를 자동화하면서도, 답을 모를 때는 적절한 부서로 연결하는 현실적인 상담 구조를 구현해 서비스 신뢰도를 높였습니다.

직무 연결 포인트

트렌드를 빠르게 따라가는 것보다, 실제 서비스 요구사항에 맞게 구조를 다시 설계하는 판단력이 중요하다는 점을 보여주는 사례입니다. PTKOREA의 AI 뉴스레터, 트렌드 관심도와도 자연스럽게 연결됩니다.

STAR 요약
S

T
목표는 실시간 음성 흐름을 끊지 않으면서도, 문법 교정·표현 교정·사고 확장 등 여러 도움 기능을 함께 제공하는 영어 회화 서비스를 만드는 것이었습니다.

실시간 음성 서비스에서 응답 지연이 사용자 경험을 저해한다고 판단하고, Agent 호출 구조를 유지하기보다 비동기 파이프라인으로 전환해 안정성과 응답 속도를 개선했습니다. Nova의 대화 내역을 DynamoDB에 실시간 로깅하고, Lambda가 이를 가져와 각 도움 기능에 비동기적으로 전달한 뒤 응답은 화면으로 분리 표시하도록 재설계했습니다.

캐릭터 기반 영상 생성 파이프라인

캐릭터 일관성을 핵심 문제로 두고, 3D 검토 후 실제 적용 가능한 구조로 재설계한 콘텐츠 자동화 프로젝트

직무 연결 포인트

PTKOREA처럼 콘텐츠와 마케팅 활용성을 중시하는 환경에서 특히 연결되는 프로젝트입니다. 단순 생성 모델 적용이 아니라 브랜드 캐릭터 일관성, 빠른 제작, 다양한 장면 구성을 동시에 만족시키는 제작 자동화 구조를 고민했다는 점이 핵심입니다.

STAR 요약
S
기업이나 서비스가 이미 보유한 캐릭터를 활용해 영상을 만들려면, 보통 캐릭터 일관성과 제작 속도 사이에서 타협이 필요했습니다. 특히 짧은 홍보 영상도 장면 구성이 단조로우면 결과물이 쉽게 지루해질 수 있어, 일관성과 다양성을 함께 확보하는 구조가 중요했습니다.
T
캐릭터 일관성을 핵심 문제로 정의하고, 3D 기반 접근을 포함한 여러 방식을 검토한 뒤 실제 서비스 환경에서 적용 가능한 구조로 재설계하는 것이 목표였습니다. 동시에 매번 다른 캐릭터가 들어와도 추가 학습 없이 대응할 수 있도록 latency와 운영성도 함께 고려했습니다.
